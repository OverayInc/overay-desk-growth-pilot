# UIUX 사용 시나리오와 개선 항목

## 시나리오 1. 아침에 전체 게임 포트폴리오 상태를 본다

사용자 목표:

- 10개 내외 게임 중 어느 게임이 움직이고 있는지 빠르게 파악한다.
- 위시리스트, 구매, 매출, 크리에이터 풀을 게임별로 비교한다.
- 바로 특정 게임으로 좁혀서 본다.

기존 문제:

- 특정 단일 제품 중심의 화면처럼 보였다.
- 게임 카드가 필터나 작업 흐름과 연결되어 있지 않았다.
- 주요 숫자의 위계가 약하고 카드 간 비교가 어렵다.

개선:

- Portfolio Overview를 첫 섹션으로 유지하되 카드 자체를 선택 가능한 필터로 변경한다.
- 선택된 게임을 상단 scope와 카드 상태로 동시에 표시한다.
- 카드 안에 위시리스트, 구매, 매출, 크리에이터 수를 같은 위치에 배치한다.

## 시나리오 2. 특정 게임의 오늘 지표를 확인한다

사용자 목표:

- 전체 또는 개별 게임의 오늘/최근 7일 성과를 본다.
- 성과 좋은 캠페인과 연락 우선순위를 같은 화면에서 확인한다.

기존 문제:

- 상단 필터와 지표 영역의 관계가 약했다.
- 게임을 바꿨을 때 현재 화면 범위가 명확하지 않았다.
- 보조 패널이 카드 안 카드처럼 보여 시선이 분산됐다.

개선:

- 현재 scope를 Today heading에 명확히 표시한다.
- metric card의 숫자, 보조 지표, 색상 규칙을 정리한다.
- 캠페인 성과와 연락 우선순위를 동일한 작업 밴드 안에서 균형 있게 배치한다.

## 시나리오 3. 새 게임/캠페인을 등록한다

사용자 목표:

- 새 게임을 추가하고 곧바로 캠페인을 연결한다.
- 필수 입력만 빠르게 채운다.

기존 문제:

- 입력 폼이 항상 노출되어 목록 탐색을 방해했다.
- 게임 선택 필드가 많아 반복 작업처럼 느껴졌다.
- 등록 성공 후 어떤 scope에 저장됐는지 체감이 약했다.

개선:

- 생성 폼을 접이식 action panel로 전환한다.
- 현재 선택된 게임을 폼의 기본값으로 사용한다.
- 저장 후 목록과 카운트가 바로 갱신되도록 유지한다.

## 시나리오 4. 크리에이터 연락 우선순위를 관리한다

사용자 목표:

- 어느 게임에 어떤 크리에이터가 붙어 있는지 본다.
- 적합도와 상태 기준으로 연락할 대상을 고른다.

기존 문제:

- 태그와 상태가 테이블에서 평평하게 보였다.
- 전체 게임 보기에서 크리에이터가 어느 게임 소속인지 한눈에 약했다.
- 긴 이름/이메일/UTM이 레이아웃을 흔들 수 있었다.

개선:

- 게임명을 명확한 테이블 컬럼과 queue tag로 표시한다.
- 상태 badge 색상을 작업 상태별로 강화한다.
- 긴 텍스트는 줄바꿈과 최대폭을 안정적으로 처리한다.

## 시나리오 5. Steam Key를 배포하고 기록한다

사용자 목표:

- 수신자, 게임, 상태, 마스킹된 키를 빠르게 확인한다.
- 키 원문 노출 없이 배포 기록만 남긴다.

기존 문제:

- 키 입력 폼이 항상 보여 보안적으로도 시각적으로도 부담스러웠다.
- 테이블의 중요한 값과 보조 값의 구분이 약했다.

개선:

- 키 입력은 접이식 panel 안에 둔다.
- 원문 키는 계속 API 응답에 노출하지 않고 마스킹 값만 표시한다.
- 상태와 게임 소속을 먼저 읽히게 한다.

## 시나리오 6. Steamworks CSV와 UTM 링크로 성과를 연결한다

사용자 목표:

- CSV를 업로드해 게임/캠페인별 지표를 반영한다.
- 크리에이터별 UTM 링크를 빠르게 생성한다.

기존 문제:

- CSV와 UTM이 별도 작업처럼 보이고 화면 하단에 묻혔다.
- 결과 링크가 생성되어도 후속 행동이 약했다.

개선:

- CSV와 UTM 영역을 작업 도구처럼 배치한다.
- UTM 결과를 별도 결과 박스로 강조한다.
- 선택된 게임의 Steam URL을 기본값으로 활용한다.
